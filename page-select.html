Usage

Single Select

It gives users the ability to choose a single item from a list of dropdown menu items. It's usually found in forms or inline edit.

Anatomy

  1. Label: Labels should be sentence case as short as possible while clearly explaining the content of the menu.
  2. Menu: Contains the list of all selectable items. These can be grouped into sub-categories under headings. Each item can also contain an icon or color.

Multi Select

It gives users the ability to choose multiple items from a list of dropdown menu items. It's usually found in forms or inline edit.

Anatomy

  1. Text input field: The field where users click in to receive the suggestions they can choose from. Once selected, the user's choices are then shown in the field.
  2. Menu: Contains the list of all selectable items. These can be grouped into sub-categories under headings.
  3. Tag: Tags show what options have been selected, and can be removed by selecting the adjacent "x" (Clear all).
  4. Clear all "x": This clears everything in the text input field. It only appears when a user has selected at least one option.

Smart Search

It gives users the ability to type, search and choose a single item from a list of dropdown menu items. It's usually found in forms.

Anatomy

  1. Text input field: The field where users type in to receive the suggestions they can choose from. Once filled with 2 characters at least, the user's choices are then shown in the field.
  2. Menu: Contains the list of all selectable items. These can be grouped into sub-categories under headings.
  3. Tag: Tag shows the selected option, and can be removed by selecting the adjacent "x" (Clear all).
  4. Clear all "x": This clears everything in the text input field. It only appears when a user has selected an option.

Best Practices

  1. Order your list of options in a way that will make the most sense to the user.
  2. Remove options that rarely get used.
  3. Don't overwhelm the user with too many options.

Content Guidelines

  1. Use concise and descriptive labels so that users clearly know the purpose of the selection. For example, use a "Save to..." label for a list containing options such as "Cloud, Local drive, CD".
  2. Keep options to a single line of text.
  3. Use commonly known terms for your options, make sure they're clear and direct.

JavaScript Instantiation

This is the basic configuration for each variation of the select component.

Default

$( '#unique-id' ).SUIselect2();

Dropdown option with icon

$( '#unique-id' ).SUIselect2({
	templateResult: SUI.select.formatIcon,
	templateSelection: SUI.select.formatIconSelection,
	escapeMarkup: function( markup ) {
		return markup;
	}
});

Dropdown option with color

$( '#unique-id' ).SUIselect2({
	templateResult: SUI.select.formatColor,
	templateSelection: SUI.select.formatColorSelection,
	escapeMarkup: function( markup ) {
		return markup;
	}
});

Smart Search

$( '#unique-id' ).SUIselect2({
	minimumInputLength: 2,
	maximumSelectionLength: 1
});

By adding sui-select class there's no need for any JavaScript instantiation but if you need to pass data through JS or any other configuration, you can do it by using .SUIselect2( 'destroy' ) before calling .SUIselect2().

Configuration

To configure custom options when you initialize SUIselect2, simply pass an object in your call to .SUIselect2().

JS options

This is a list with the most common SUIselect2 configuration options.

Option Type Default Description
dropdownParent jQuery selector or DOM node $( document.body ) Allows you to modify where dropdown is going to be placed. Use this option when select is placed inside modals.
templateResult callback Customizes the way that dropdown options are rendered. You can select between: SUI.select.formatIcon to show an icon or SUI.select.formatColor to show a color square.
templateSelection callback Customizes the way that selections are rendered. You can select between: SUI.select.formatIconSelection to show an icon or SUI.select.formatColorSelection to show a color square.
escapeMarkup callback Utils.escapeMarkup Handles automatic escaping of content rendered by custom templates.

Since SUIselect2 is based on Select2, you can find more configuration options listed here.

data-* attributes

It is recommended that you declare your configuration options by passing in an object when initializing SUIselect2. However, you may also define your configuration options by using the HTML5 data-* attributes, which will override any options set when initializing SUIselect2 and any defaults.

Attribute Default Description
data-theme Enables markup and styles for different select styles. Use icon, color or search value depending the case. Make use of Examples tab to confirm the correct markup.
data-placeholder Make use of this attribute along with an empty <option> at the beginning of the options' list.
data-width Include any fixed value you need for the selector, for example 200px. This attribute works with single select only.
data-height Use this attribute with sm value to make selector height 30px. This attribute works with single select only.
data-search false Use true value when you need to enable search input on dropdown list. This attribute works with single select only.

Classes

Single select element by default occupies a single row and the full width of it, but there are times when you need to use it as an inline element. For that, you need to add sui-select-inline class to select element.
