Documentation

Simple Tree Find here the basics to make SUI Tree work correctly on your installation.
1. Initialize JS
Initialize JS 1. Get latest stable Shared UI release. 2. Load shared-ui.js on your plugin or trees.js file if you are using individual components. 3. SUI.suiTree() will initialize auto-magically on load, but if this doesn't happen you can initialize it on your installation by running the following script: 
$( function() {

	// Init SUI Tree
	if ( 0 !== $( 'SUI_BODY_CLASS .sui-tree' ).length ) {

		$( 'SUI_BODY_CLASS .sui-tree' ).each( function() {
			SUI.suiTree( $( this ), false );
		});
	}

}( jQuery ) );
2. Markup Selectors Tree
Starter Markup Here you will find two different base markups that will work with Simple Tree. Click on each tab to see more details. NOTE: Please pay attention to the notes I left in the sample code.
Basic Mode This markup uses only <span />'s inside <li /> reason why elements like checkboxes or buttons are handled with roles to make them correctly accessible for screen readers. 
<ul class="sui-tree" data-tree="selector" role="group">

	<li role="treeitem">

		<span class="sui-tree-node">

			<!-- Checkbox -->
			<span class="sui-node-checkbox" role="checkbox">
				<span class="sui-screen-reader-text">Select this item</span>
			</span>

			<!-- Text -->
			<span class="sui-node-text">Parent / Group</span>

			<!-- Button expander (for open/close) -->
			<span role="button" data-button="expander">
				<span class="sui-screen-reader-text">Open or close this item</span>
			</span>

		</span>

		<!-- NOTE:
		Wrap more items inside UL to create a new group inside parent item
		-->
		<ul role="group">

			<li role="treeitem">

				<span class="sui-tree-node">

					<!-- Checkbox -->
					<span class="sui-node-checkbox" role="checkbox">
						<span class="sui-screen-reader-text">Select this item</span>
					</span>

					<!-- Text -->
					<span class="sui-node-text">Child Item</span>

					<!-- IMPORTANT:
					Notice expander button isn't here, this because it's not necessary.
					If for any case you can't remove the expander button from the markup
					don't worry because the JS function will check if item doesn't have
					child elements and will remove the expander button.
					-->

				</span>

			</li>

		</ul>

	</li>

</ul>

If you need to get the value assigned to checkbox, that must be retrieved using JS. For example, assign data-attribute="..." and check on change if li has aria-selected="true" attribute.

4. Tricks
1. Expand Item Add aria-expanded="true" to show parent item child elements on load. 
<li role="treeitem" aria-expanded="true"> ... </li>
2. Select Item Add aria-selected="true" to check an item on load. 
<li role="treeitem" aria-selected="true"> ... </li>
3. Disable Item Add aria-disabled="true" to disable an item and its childs. 
<li role="treeitem" aria-disabled="true"> ... </li>
4. Change Item Icon Add data-type=" ... " to change an item icon. Don't forget to replace ... with the type of icon you want to use.

Fancy Tree Find here the basics to make Fancy Tree JS with SUI styles work in your plugin.
1. Installation
Installation 1. Get latest Fancytree JS. Read their Wiki for correct steps. 2. Init Fancytree JS on your plugin: 
$( function() {

	// Init fancy tree
	$( '#tree-id' ).fancytree({
		// Put here all your settings
	});

}( jQuery ) );

In the showcase we use Fancytree JS 2.27.0 because latest available version 2.30.2 has an annoying bug that makes page scroll up or down when you click on an item.

2. Markup
Markup Selector Notice the class and attribute to load the correct styles for your tree. 
<div
	id="tree-id"
	class="sui-tree-fancy"
	data-tree="selector"
></div>
Markup Directory Browser Notice the class and attribute to load the correct styles for your tree. 
<div
	id="tree-id"
	class="sui-tree-fancy"
	data-tree="directory"
></div>
3. Tricks Fancytree JS
1. Enable / Disable Checkbox You can do this globally (for all checkboxes on tree) or per item. 
$( function() {

	$( '#tree-id' ).fancytree({

		// (Bool) global checkbox
		checkbox: false,

		// (Bool) Checkbox per item
		source: [
			{ title: 'Item', checkbox: true }
		],
	});

}( jQuery ) );
2. Expand Item Some times you need an item expanded and show its child items as soon as tree loads. 
$( function() {

	$( '#tree-id' ).fancytree({

		source: [
			{
				title: 'Item',
				expanded: true, // Expand item on load
				child: [ { title: 'Child Item' } ] // IMPORTANT: For expand to work correctly make sure you have child items
			}
		],
	});

}( jQuery ) );
3. Custom Icon When you use Directory Browser tree mode, notice each item have an icon (can be a folder or a page). To change the item icon follow this code example. 
$( function() {

	$( '#tree-id' ).fancytree({

		// Replace icon per item
		source: [
			{
				title: 'Item',
				icon: 'sui-icon-photo-picture' // Replace this with any SUI icon class
			}
		],

		// Replace folder icon (globally)
		icon: function( event, data ) {

			if ( data.node.isFolder() ) {
				return 'sui-icon-folder'; // Replace this with any SUI icon class.
			}
		},
	});

}( jQuery ) );
4. Remove Icons When you use Selectors tree mode, notice items don't have an icon. By default Fancytree JS enables these icons, you need to remove them to make it match the design. 
$( function() {

	$( '#tree-id' ).fancytree({

		// Remove icons (globally)
		icon: false,

	});

}( jQuery ) );
